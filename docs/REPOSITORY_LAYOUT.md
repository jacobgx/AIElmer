# Repository Layout

This repository is organized as an AI-ready ElmerFEM modeling knowledge-base framework.

```text
.
├── README.md
├── knowledge/
│   ├── README.md
│   ├── ELMER_AI_MODELING_STANDARD.md
│   ├── ELMER_KB_DISTILLATION_PLAN.md
│   ├── ELMER_QUICKREF.md
│   ├── CODING_STANDARD.md
│   ├── CASE_CYLINDER_FLOW.md
│   └── distilled/
├── templates/
│   └── elmer_case/
├── scripts/
├── elmer/
└── docs/
```

## Publishable Core

These paths are intended for a public GitHub repository:

- `README.md`
- `LICENSE.md`
- `.gitignore`
- `.gitattributes`
- `docs/`
- `knowledge/`
- `templates/`
- `scripts/query_kb.ps1`
- `scripts/new_elmer_case.ps1`
- `scripts/check_elmer_case.ps1`
- `scripts/prepare_github_share.ps1`
- `elmer/README.md`

## Local-Only Artifacts

These paths are useful during modeling but should not be committed by default:

- `tmp/`
- `build/`
- `results/`
- `elmer/*/mesh/`
- `elmer/*/logs/`
- `elmer/*/results_raw/`
- `*.mph`
- `*.vtu`
- `*.pvd`
- `*.result`
- `*.gif`

## Case Directories

Each user-created model should live under `elmer/<case_id>/` and follow the contract generated by:

```powershell
.\scripts\new_elmer_case.ps1 -Name heat_exchanger
```

The case contract is documented in `knowledge/ELMER_AI_MODELING_STANDARD.md`.
